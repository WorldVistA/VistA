What's New in CPRS GUI v.27 (OR*3.0*243)

CCOW (Context Management)

·      Joining Context and Breaking Context  Added notes to let users know that the Rejoin Context menu item will not be available after a context error occurs until the user logs out and then back into CPRS.

 

CPRS Dates and Times

·      Entering Dates and Times into CPRS  This topic tells users how dates and time can be entered into CPRS.

 

Font Sizes in CPRS

·      Changing the Font Size  This topic now includes the advice that for font sizes larger than 18 point, user should take advantage of magnification software.

 

Graphing

·      Graphing (overview)  Added some changes to definitions found on this page. References to comments and free-text on the graphs.

·      Graphing: Starting CPRS Graphing  updated the graphic and added some information to this page.

·      Graphing: Exporting Data  This feature enables users to export graphing data to an Excel spreadsheet automatically.

 

Health Summary

·      Viewing a Health Summary  Included the errors that display when the HDR cannot be reached. CPRS will also display error text after the name of facilities listed when the user selects the Remote Data button when it is active (blue text displayed).

 

Infusion Orders

·      Infusion Orders Overview  To address a patient safety issue, CPRS changed to use the term Infusions for various orders that used to be under IV Fluids because some of these items were not given intravenously. This section introduces the idea of continuous and intermittent infusion orders.

·      Placing Continuous Infusion Orders  This topic tells users how to order continuous infusion orders.

·      Placing Intermittent Infusion Orders  This topic tells users how to order intermittent infusion orders.

 

Inpatient Medications

·      Ordering Inpatient Medications (simple dose)  made several changes.

 

Medications Tab

·     Sorting the Medications View  New methods of sorting Medication information on the Meds tab.

·     Other Medication Actions  This topics tells users how the right-click action on the Meds tab works when the user right-clicks and items are highlighted or when no items are highlighted. There is a note also about and "invalid pharmacy number" error that occurs infrequently.

 

Mental Health Assessments Requirements for a User

·      Using Mental Health Assessments in CPRS  Details what must be in place to make use of the new Mental Health items.

 

New Blood Bank Software (VBECS ordering)

·      Blood Products (Overview)  CPRS has introduced a new dialog that will be activated by another patch to enable the users to order blood products and diagnostic tests. This topic describes the new dialog.

·      Ordering Blood Components and Diagnostic Tests  Steps on how to order these items.

 

Notifications

·      Processing Notifications  Added a note about what happens when a user tries to process notifications from patients with sensitive records.

 

Patient Inquiry

·      Additional Patient Information  Next of kin and cell phone information are now in the demographic information available in CPRS.

 

Patient Record Flags

·      Viewing Patient Record Flags  Category I Patient Record Flags display with different colors, text size, and blinks. The changes are meant to differentiate Category I flags from Category II flags.

 

Patient Selection

·      Selecting a Patient  Added a note that when a user is typing in the patient's name or social security number, CPRS will not automatically select a patient name until a unique match is found.

 

Refresh Patient Information (Encounter Location Switching)

·      Refresh Patient Information  Added a brief description of a new dialog that enables users to choose where the orders should be created if the patient moves while the chart is open, but the user has not written orders yet.

 

Remote Data

·      Choosing RDV or VistaWeb to View Remote Data  How the user chooses which method to choose has changed. Now the user will choose each time because two buttons now exist.

·      Remote Data  New screen capture showing buttons and changes to reflect the new buttons.

·      Remote Order Checking  A change was made to the message that informs the user the check on done on local data only.

·      Viewing a Report  Slight change to instructions because of changes to the buttons.
